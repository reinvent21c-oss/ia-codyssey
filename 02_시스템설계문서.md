# 시스템 설계 문서

본 문서는 B2B 제조기업 홈페이지 리뉴얼 킥오프 회의록 자동 정리 프롬프트 패키지의 시스템 설계 문서입니다.

세부 작성 내용은 `evidence` 폴더에 단계별로 정리했습니다.

## 구성

1. 문제 정의: `evidence/9-1_문제정의기초정리.md`
2. 입력 템플릿: `evidence/9-2-1 입력 템플릿(재사용 가능).rtf`, `evidence/9-2-2. 이번 과업 적용 예시.rtf`
3. 페르소나 정의: `evidence/9-3. 페르소나 정의.rtf`
4. 시스템 프롬프트 v1: `evidence/9-4. 시스템 프롬프트 v1 초안.rtf`
5. Few-shot 예시 3개: `evidence/9-5. Few-shot 예시 3개.rtf`
6. 환각 검증 설계: `evidence/9-6. 환각 검증 설계.rtf`
7. v1 실행 결과: `evidence/9-7-1. v1_Claude_회의록정리_실행결과.pdf`
8. 프롬프트 개선 이력(v1 → v2): `evidence/9-7-2. 프롬프트 개선 이력(v1 → v2).rtf`
9. 최종 프롬프트 전문: `evidence/9-8. 최종 프롬프트 전문 정리.rtf`

## 현재 상태

9단계 시스템 설계 문서 작성은 완료했으며, 다음 단계에서는 선택 모델인 Claude Opus 4.8을 기준으로 10턴 이상 실행 로그를 작성할 예정입니다.
