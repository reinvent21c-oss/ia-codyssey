# 시스템 설계 문서

본 문서는 B2B 제조기업 홈페이지 리뉴얼 킥오프 회의록 자동 정리 프롬프트 패키지의 시스템 설계 요약 문서입니다.

세부 작성 내용은 `evidence/` 폴더에 단계별로 정리했습니다.

---

## 구성

1. 문제 정의: `evidence/9-1_문제정의기초정리.md`
2. 입력 템플릿: `evidence/9-2-1 입력 템플릿(재사용 가능).rtf`, `evidence/9-2-2. 이번 과업 적용 예시.rtf`
3. 페르소나 정의: `evidence/9-3. 페르소나 정의.rtf`
4. 시스템 프롬프트 v1: `evidence/9-4. 시스템 프롬프트 v1 초안.rtf`
5. Few-shot 예시 3개: `evidence/9-5. Few-shot 예시 3개.rtf`
6. 환각 검증 설계: `evidence/9-6. 환각 검증 설계.rtf`
7. v1 실행 결과: `evidence/9-7-1. v1_Claude_회의록정리_실행결과.pdf`
8. 프롬프트 개선 이력(v1 → v2): `evidence/9-7-2. 프롬프트 개선 이력(v1 → v2).rtf`
9. 최종 프롬프트 전문: `evidence/9-8. 최종 프롬프트 전문 정리.rtf`

---

## 설계 목적

이번 시스템 설계의 목적은 회의 메모를 단순 요약하는 것이 아니라, 웹에이전시 PM이 실제 업무에서 사용할 수 있는 회의록 초안을 안정적으로 생성하는 것입니다.

특히 다음 항목을 분리하는 데 중점을 두었습니다.

- 결정사항
- 후속 실행 항목
- 확인된 사실
- 가능한 리스크
- 확인 필요 사항
- 후속 일정

---

## 최종 적용 모델

최종 적용 모델은 `Claude Opus 4.8`입니다.

1회차에서는 ChatGPT 5.5, Claude Opus 4.8, Gemini 3.5 Flash를 비교했고, 2회차에서는 비교 공정성 보완을 위해 Gemini 3.1 Pro를 추가 테스트했습니다.

2회차 보완 후에도 이번 과업의 핵심인 확인 필요 항목 분리와 환각 억제 측면에서 Claude Opus 4.8이 가장 안정적이라고 판단했습니다.

---

## 현재 상태

9단계 시스템 설계 문서 작성은 완료했습니다.

또한 최종 적용 모델인 Claude Opus 4.8을 기준으로 10턴 이상 실행 로그를 작성했으며, 해당 결과는 루트의 `03_실행로그.md`와 `logs/` 폴더에 정리했습니다.

2회차에서는 Gemini 3.1 Pro 재테스트 결과를 반영하여 모델 비교의 공정성을 보완했습니다.
