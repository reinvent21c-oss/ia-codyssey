# B1-1 미션 제출 자료

## 1. 미션 개요

이번 미션에서는 웹에이전시의 **B2B 제조기업 홈페이지 리뉴얼 킥오프 회의록 정리 업무**를 대상으로, LLM 기반 업무 자동화 프롬프트 패키지를 설계했습니다.

핵심 과업은 비정형 회의 메모에서 다음 내용을 분리하여 실무 문서로 정리하는 것입니다.

* 결정사항
* 후속 실행 항목
* 확인된 사실
* 가능한 리스크
* 확인 필요 사항
* 후속 일정

이 과업은 단순 요약이 아니라, 회의 이후 프로젝트 진행에 필요한 정보를 누락 없이 정리하고, 확정된 내용과 미확정 내용을 구분하는 데 목적이 있습니다.

## 4. 모델 비교 개요

1회차에서는 다음 3개 모델을 비교했습니다.

| 모델               | 비교 목적                       |
| ---------------- | --------------------------- |
| ChatGPT 5.5      | 구조화 및 실무 문서화 품질 확인          |
| Claude Opus 4.8  | 결정사항/미확정 사항 분리 및 긴 문맥 유지 확인 |
| Gemini 3.5 Flash | Gemini 계열 모델의 기본 응답 품질 확인   |

1회차 평가 결과, 이번 과업에서는 **Claude Opus 4.8**이 결정사항, 검토사항, 확인 필요 사항을 가장 안정적으로 분리한다고 판단했습니다.

다만 1회차에서는 Gemini만 Flash 계열 모델을 사용했기 때문에, ChatGPT 5.5 및 Claude Opus 4.8과 비교했을 때 모델 등급 차이가 있었습니다. 이로 인해 모델 비교의 공정성 측면에서 한계가 있음을 확인했습니다.

## 5. 2회차 보완 내용

2회차에서는 모델 비교의 공정성을 보완하기 위해 **Gemini 3.1 Pro**를 추가 테스트했습니다.

Gemini 3.1 Pro 테스트는 기존 Gemini 3.5 Flash 대화의 영향을 줄이기 위해 새 대화창에서 진행했습니다. 또한 이전 Gemini 결과, Claude 결과, ChatGPT 결과를 참고하지 않고, 동일한 회의 메모와 동일한 작성 규칙만 기준으로 테스트했습니다.

2회차에서도 평가 기준은 1회차와 동일하게 유지했습니다.

| 평가 기준           | 확인 내용                               |
| --------------- | ----------------------------------- |
| 정확성             | 회의 메모의 사실관계를 왜곡하지 않고 정리했는가          |
| 형식 준수           | 요청한 출력 항목과 Markdown 형식을 지켰는가        |
| Action Items 품질 | 담당 역할, 해야 할 일, 기한/상태를 실행 가능하게 정리했는가 |
| 확인 필요 항목 분리     | 미정 사항과 추가 확인이 필요한 내용을 별도로 분리했는가     |
| 환각 억제           | 회의 메모에 없는 내용을 확정적으로 작성하지 않았는가       |
| 실무 활용성          | 웹에이전시 PM이 회의록 초안으로 바로 검토·활용할 수 있는가  |

## 6. 2회차 모델 비교 결과 요약

Gemini 3.1 Pro는 기존 Gemini 3.5 Flash보다 문서 구조화, 형식 준수, Action Items 정리 측면에서 개선된 결과를 보였습니다.

특히 7개 출력 항목을 모두 구성했고, 후속 실행 항목도 담당 역할, 작업, 기한 중심의 표로 정리했습니다.

다만 이번 과업의 핵심은 단순히 보기 좋은 회의록을 작성하는 것이 아니라, **확정된 결정사항과 확인 필요 항목을 안정적으로 분리하고, 회의 메모에 없는 내용을 확정적으로 작성하지 않는 것**이었습니다.

Gemini 3.1 Pro 결과에서는 일부 검토성 항목이 결정사항처럼 강하게 표현된 부분이 있어 최종 공유 전 PM 검토가 필요하다고 판단했습니다.

따라서 2회차에서는 Gemini 3.1 Pro 테스트를 통해 모델 비교의 공정성을 보완하되, 최종 적용 모델은 **Claude Opus 4.8**로 유지했습니다.

---

## 7. 최종 모델 선정

최종 적용 모델은 **Claude Opus 4.8**입니다.

선정 이유는 다음과 같습니다.

* 결정사항과 미확정 사항을 가장 안정적으로 분리했습니다.
* 확인 필요 사항을 확정된 내용처럼 작성하는 위험이 상대적으로 적었습니다.
* 후속 실행 항목의 선후관계와 담당 역할을 잘 유지했습니다.
* 조건 변경과 추가 정보 제공 상황에서도 문맥 유지가 안정적이었습니다.
* 사내 공유용 회의록 또는 클라이언트 전달용 회의록으로 다듬기 쉬웠습니다.

단, 이 결론은 Claude가 모든 업무에서 절대적으로 우수하다는 의미가 아닙니다.
이번 **B2B 제조기업 홈페이지 리뉴얼 킥오프 회의록 요약 및 후속 실행 항목 정리** 과업에서 가장 안정적으로 작동했다는 의미입니다.
