- 클라이언트 유형:
- 회의 유형:

- 참석 역할:

### 2. 작성 목적
- 이번 회의록 정리의 목적:
  예) 사내 공유용 회의록 작성 / 후속 실행 항목 정리 / 클라이언트 전달용 요약본 작성

### 3. 공유 대상
- 공유 대상:
  예) 내부 프로젝트팀 / 클라이언트 / 경영진 / 외부 협력사

### 5. 필수 출력 항목
다음 항목으로 구분해 작성한다.

1. 회의 목적
2. 결정사항
3. 후속 실행 항목
4. 확인된 사실
5. 가능한 리스크
6. 확인 필요 사항
7. 후속 일정

### 6. 작성 규칙
- 회의 메모에 명시된 내용만 확정 표현으로 작성한다.
- 확정되지 않은 의견, 제안, 추정은 결정사항으로 작성하지 않는다.
- 담당자, 기한, 선행 조건이 불명확한 업무는 확인 필요 사항으로 분리한다.
- 후속 실행 항목은 담당 역할 / 해야 할 일 / 기한 또는 조건 / 비고로 정리한다.
- 인물 실명은 사용하지 않고 역할명으로 작성한다.
- 사내 공유용 문체로 간결하고 명확하게 작성한다.

### 7. 확인 질문 기준
다음에 해당하면 임의로 작성하지 말고 확인 필요로 분리한다.

- 담당자가 불명확한 경우
- 기한이 명시되지 않은 경우
- 결정사항인지 의견인지 구분이 어려운 경우
- 클라이언트 회신 대기 중인 경우
- 추가 견적, 범위 변경, 일정 변경 가능성이 있는 경우

### 8. 금지 사항
- 회의 메모에 없는 내용을 만들어내지 않는다.
- 미확정 사항을 결정사항처럼 쓰지 않는다.
- 내부 리스크 판단을 클라이언트 확정 문구처럼 쓰지 않는다.
- 후속 실행 항목을 누락하지 않는다.
