# Gemini 결과 — B2B 제조기업 홈페이지 리뉴얼 프로젝트 킥오프 회의록

## 1. 회의 목적

* 기존 홈페이지의 모바일 가독성 저하 및 제품 문의 전환율 낮음 문제를 개선하기 위해 진행한 회의입니다.
* 1차 목표를 수립했습니다.

  * 회사 신뢰도 개선
  * 제품 정보 탐색 편의성 향상
  * 문의 접수 증가
* 주요 메뉴 구성 및 핵심 기능 범위를 논의했습니다.
* 향후 일정 및 역할별 작업을 조율했습니다.

---

## 2. 결정사항

| 항목            | 결정 내용                               |
| ------------- | ----------------------------------- |
| 홈페이지 주요 메뉴 구성 | 회사소개, 제품소개, 기술자료, 납품사례, 고객문의        |
| 제품소개 페이지 구성   | 제품군별 목록, 상세페이지, 인증서 다운로드 영역 포함      |
| 고객문의 페이지 구성   | 이름, 연락처, 이메일, 문의 제품군, 문의 내용 입력 폼 구축 |
| 고객문의 연동 기능    | 문의 폼 접수 내역 이메일 발송 기능을 기본 범위에 포함     |
| 관리자 기능 1차 범위  | 제품 정보 수정 기능만 우선 포함                  |
| 인증서 파일 관리 기능  | 추가 견적 대상으로 분리                       |
| 퍼블리싱 작업 기준    | 반응형 기준으로 작업하되 모바일 화면을 우선 고려         |

## 4. 확인된 사실

* 기존 홈페이지는 모바일 화면 가독성이 떨어지고 제품 문의 전환율이 낮은 상태입니다.
* 클라이언트는 관리자 페이지를 통해 제품 정보를 직접 수정하기를 원합니다.
* 개발 담당 의견에 따르면 제품 정보 관리 기능은 가능합니다.
* 인증서 파일 업로드 관리까지 포함하면 개발 범위와 일정이 늘어날 수 있습니다.
* 기존 홈페이지의 게시판 데이터 양과 구조는 현재 확인되지 않았습니다.
* 데이터 이전 여부는 확정되지 않았습니다.
* 다국어 페이지 필요성에 대한 의견은 나왔으나, 구체적인 언어 범위와 번역 방식은 정해지지 않았습니다.

## 5. 가능한 리스크

| 리스크             | 설명                                                                         |
| --------------- | -------------------------------------------------------------------------- |
| 자료 전달 지연        | 회사 로고, 카탈로그, 인증서, 납품사례 등 클라이언트 자료 전달이 지연될 경우 전체 일정에 영향을 줄 수 있습니다.          |
| 인증서 관리 기능 추가    | 인증서 관리 기능이 추가 범위로 확정될 경우 최종 일정 및 개발 범위가 늘어날 수 있습니다.                        |
| 게시판 데이터 이전 불확실성 | 기존 홈페이지 게시판 데이터의 양이 많거나 구조가 복잡할 경우 예기치 못한 리소스가 소요될 수 있습니다.                 |
| 다국어 페이지 범위 미확정  | 다국어 페이지 도입 여부, 언어 범위, 번역 방식이 뒤늦게 확정될 경우 기획, 디자인, 개발 전 과정의 작업량이 증가할 수 있습니다. |
