# Claude 결과 — B2B 제조기업 홈페이지 리뉴얼 프로젝트 킥오프 회의록

## 회의 정보

| 항목    | 내용                                                      |
| ----- | ------------------------------------------------------- |
| 회의 주제 | B2B 제조기업 홈페이지 리뉴얼 프로젝트 킥오프 회의                           |
| 참석 역할 | 클라이언트 담당자 / 프로젝트 매니저 / 기획 담당 / 디자인 담당 / 퍼블리싱 담당 / 개발 담당 |

---

## 1. 회의 목적

기존 홈페이지의 노후화로 인한 문제를 개선하기 위해 홈페이지 리뉴얼 프로젝트의 착수 및 1차 범위 합의를 진행했습니다.

주요 문제는 다음과 같습니다.

* 모바일 화면 가독성 저하
* 제품 문의 전환율 저조

리뉴얼 1차 목표는 다음 세 가지로 정리되었습니다.

* 회사 신뢰도 개선
* 제품 정보 탐색 편의성 향상
* 문의 접수 증가

---

## 2. 결정사항

* 홈페이지 주요 메뉴를 **회사소개 / 제품소개 / 기술자료 / 납품사례 / 고객문의**로 구성하기로 했습니다.
* **제품소개 페이지**는 제품군별 목록, 상세페이지, 인증서 다운로드 영역을 포함하기로 했습니다.
* **고객문의 페이지**는 이름, 연락처, 이메일, 문의 제품군, 문의 내용을 입력받는 폼으로 구성하기로 했습니다.
* 관리자 페이지의 **1차 개발 범위는 제품 정보 수정 기능만 포함**하고, **인증서 파일 관리 기능은 추가 견적 대상으로 분리**하기로 했습니다.
* 다음 회의는 사이트맵 초안 도출 이후 진행하기로 했습니다.

> 참고: 퍼블리싱 담당이 제시한 “반응형 기준 + 모바일 우선 작업” 방향은 회의 중 제시된 작업 의견으로, 확정된 결정사항으로 명시되지는 않았습니다.

## 4. 확인된 사실

* 클라이언트가 기존 홈페이지의 모바일 가독성 저하 및 제품 문의 전환율 저조를 문제로 제기했습니다.
* 개발 담당은 제품 정보 관리 기능 구현이 가능하다고 설명했습니다.
* 개발 담당은 인증서 파일 업로드 관리까지 포함할 경우 개발 범위와 일정이 늘어날 수 있다고 설명했습니다.
* 개발 담당은 문의 폼 접수 내역을 이메일로 발송하는 기능은 기본 범위에 포함 가능하다고 설명했습니다.
* 게시판 데이터 이전 여부는 현재 데이터 양과 구조가 확인되지 않아 회의 중 즉시 확정하지 못했습니다.
* 전체 오픈 목표일은 다음 달 말로 논의되었으나, 최종 일정은 자료 전달일과 추가 기능 범위 확정 후 재산정하기로 했습니다.
* 다국어 페이지 필요성에 대한 의견이 제시되었으나, 언어 범위와 번역 방식은 이번 회의에서 정하지 않았습니다.
