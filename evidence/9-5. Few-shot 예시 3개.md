## 9-5. Few-shot 예시

Few-shot 예시는 AI가 회의록 정리 업무에서 어떤 기준으로 판단해야 하는지 보여주기 위한 예시입니다.  
본 과업에서는 실제 업무에서 발생할 수 있는 입력 유형을 3가지로 나누어 구성했습니다.

- Few-shot 1: 표준형 킥오프 회의 메모
- Few-shot 2: 확정사항과 희망사항이 혼재된 회의 메모
- Few-shot 3: 정보가 부족한 모호한 회의 메모

---

### Few-shot 1. 표준형 킥오프 회의 메모

#### 입력

[회의 메모]
- 클라이언트는 기존 홈페이지가 오래되어 기업 신뢰도가 낮아 보인다고 설명했다.
- 리뉴얼 목적은 제품 정보 정리, 문의 전환 개선, 모바일 사용성 개선이다.
- 주요 메뉴는 회사소개, 제품소개, 인증현황, 고객지원으로 구성하기로 했다.
- 클라이언트는 제품 카탈로그와 인증서 자료를 이번 주 금요일까지 전달하기로 했다.
- 디자인 담당자는 다음 주 초까지 메인 시안 방향을 제안하기로 했다.
- 개발 담당자는 문의폼 기능과 관리자 기능 범위를 검토하기로 했다.

#### 기대 출력

1. 회의 목적
- 기존 홈페이지의 노후화 문제를 개선하고, 제품 정보 전달력과 문의 전환율을 높이기 위한 홈페이지 리뉴얼 방향을 논의했습니다.

2. 결정사항
- 주요 메뉴는 회사소개, 제품소개, 인증현황, 고객지원으로 구성합니다.
- 리뉴얼의 주요 목적은 제품 정보 정리, 문의 전환 개선, 모바일 사용성 개선으로 정리합니다.

3. 후속 실행 항목

| 담당 역할 | 해야 할 일 | 기한 또는 조건 | 비고 |
|---|---|---|---|
| 클라이언트 담당자 | 제품 카탈로그 및 인증서 자료 전달 | 이번 주 금요일까지 | 콘텐츠 제작 선행 자료 |
| 디자인 담당자 | 메인 시안 방향 제안 | 다음 주 초까지 | 디자인 방향 검토 필요 |
| 개발 담당자 | 문의폼 기능 및 관리자 기능 범위 검토 | 기한 확인 필요 | 개발 범위 확정 전 검토 필요 |

4. 확인된 사실
- 기존 홈페이지는 오래되어 기업 신뢰도 측면에서 개선 필요성이 있습니다.
- 제품 정보 정리와 모바일 사용성 개선이 주요 요구사항으로 언급되었습니다.

5. 가능한 리스크
- 제품 카탈로그와 인증서 자료 전달이 지연될 경우 콘텐츠 구성 및 디자인 일정에 영향을 줄 수 있습니다.
- 개발 범위가 명확히 확정되지 않으면 견적 또는 일정 변동 가능성이 있습니다.

6. 확인 필요 사항
- 문의폼 기능과 관리자 기능의 구체적인 범위를 확인해야 합니다.
- 개발 검토 완료 시점과 최종 확정 일정을 확인해야 합니다.

7. 후속 일정
- 이번 주 금요일: 클라이언트 자료 전달 예정
- 다음 주 초: 디자인 담당자 메인 시안 방향 제안 예정

---

### Few-shot 2. 확정사항과 희망사항이 혼재된 회의 메모

#### 입력

[회의 메모]
- 클라이언트는 제품소개 페이지를 가장 중요하게 보고 있다고 말했다.
- 회사소개와 제품소개는 반드시 포함하기로 했다.
- 클라이언트는 가능하면 고객사 로고도 보여주고 싶다고 했다.
- 인증현황 페이지는 있으면 좋겠지만 자료 정리가 아직 안 되어 있다고 했다.
- 디자인은 깔끔하고 신뢰감 있는 방향을 선호한다고 말했다.
- 퍼블리싱 담당자는 모바일 우선으로 작업하는 것이 좋겠다는 의견을 냈다.
- 다국어 페이지는 예산에 따라 포함 여부를 검토하기로 했다.

#### 기대 출력

1. 회의 목적
- B2B 제조기업 홈페이지 리뉴얼의 주요 페이지 구성, 디자인 방향, 추가 기능 검토 범위를 논의했습니다.

2. 결정사항
- 회사소개와 제품소개 페이지는 홈페이지 구성에 포함합니다.
- 제품소개 페이지를 핵심 페이지로 우선 고려합니다.

3. 후속 실행 항목

| 담당 역할 | 해야 할 일 | 기한 또는 조건 | 비고 |
|---|---|---|---|
| 클라이언트 담당자 | 인증현황 자료 정리 가능 여부 확인 | 확인 필요 | 페이지 포함 여부 판단 필요 |
| 클라이언트 담당자 | 고객사 로고 공개 가능 여부 확인 | 확인 필요 | 권한 및 사용 가능 여부 확인 필요 |
| 웹에이전시 프로젝트 담당자(PM) | 다국어 페이지 포함 여부와 예산 영향 정리 | 예산 검토 후 | 범위 변경 가능성 있음 |
| 퍼블리싱 담당자 | 모바일 우선 작업 방향 검토 | 디자인 방향 확정 후 | 의견 단계로 분류 |

4. 확인된 사실
- 클라이언트는 제품소개 페이지를 중요하게 보고 있습니다.
- 회사소개와 제품소개 페이지는 포함하기로 했습니다.
- 인증현황 자료는 아직 정리되지 않았습니다.
- 다국어 페이지는 예산에 따라 포함 여부를 검토해야 합니다.

5. 가능한 리스크
- 고객사 로고 사용 가능 여부가 확인되지 않으면 디자인 구성 변경이 발생할 수 있습니다.
- 인증현황 자료가 정리되지 않으면 해당 페이지 제작 일정이 지연될 수 있습니다.
- 다국어 페이지가 포함될 경우 번역, 개발, 검수 범위가 추가될 수 있습니다.

6. 확인 필요 사항
- 고객사 로고를 실제 홈페이지에 게시할 수 있는지 확인해야 합니다.
- 인증현황 페이지를 최종 포함할지 확인해야 합니다.
- 다국어 페이지의 포함 여부와 추가 예산 여부를 확인해야 합니다.
- 모바일 우선 작업 방향이 공식 결정사항인지, 퍼블리싱 담당자의 작업 의견인지 확인해야 합니다.

7. 후속 일정
- 구체적인 일정은 회의 메모에 명시되지 않아 확인이 필요합니다.

### Few-shot 3. 정보가 부족한 모호한 회의 메모

#### 입력

[회의 메모]
- 홈페이지 리뉴얼 필요.
- 제품 페이지 중요.
- 자료는 나중에 준다고 함.
- 디자인은 깔끔하게.
- 문의 기능도 필요할 수도 있음.
- 사이트맵은 내부 제작 중이라고 했으나 전달 일정은 아직 정해지지 않음.
- 다음 주쯤 뭔가 정리하기로 함.

#### 기대 출력

1. 회의 목적
- 홈페이지 리뉴얼 필요성과 제품 페이지, 디자인 방향, 문의 기능 가능성에 대해 논의한 것으로 보입니다.
- 다만 입력 정보가 부족하여 확정된 회의 목적은 추가 확인이 필요합니다.

2. 결정사항
- 회의 메모만으로는 확정된 결정사항을 판단하기 어렵습니다.

3. 후속 실행 항목

| 담당 역할 | 해야 할 일 | 기한 또는 조건 | 비고 |
|---|---|---|---|
| 클라이언트 담당자 | 홈페이지 리뉴얼 관련 자료 전달 | “나중에”로만 언급됨 | 정확한 전달 기한 확인 필요 |
| 웹에이전시 프로젝트 담당자(PM) | 제품 페이지 중요도와 구성 범위 확인 | 확인 필요 | 핵심 페이지 여부 확인 필요 |
| 웹에이전시 프로젝트 담당자(PM) | 문의 기능 포함 여부 확인 | 확인 필요 | “필요할 수도 있음”으로 언급되어 확정 아님 |
| 웹에이전시 프로젝트 담당자(PM) | 사이트맵 확정 여부 및 전달 가능 일정 확인 | “내부 제작 중”으로만 언급됨 | 며칠 후 재확인 필요 |

4. 확인된 사실
- 홈페이지 리뉴얼 필요성이 언급되었습니다.
- 제품 페이지가 중요하다는 의견이 있었습니다.
- 깔끔한 디자인 방향이 언급되었습니다.
- 문의 기능은 가능성 수준으로 언급되었습니다.
- 사이트맵은 내부 제작 중이나 전달 일정은 정해지지 않았습니다.

5. 가능한 리스크
- 자료 전달 기한이 불명확해 기획 및 디자인 일정이 지연될 수 있습니다.
- 문의 기능 포함 여부가 확정되지 않아 개발 범위와 견적이 변동될 수 있습니다.
- 사이트맵 전달 일정이 불명확해 페이지 구조 확정이 지연될 수 있습니다.
- “다음 주쯤”이라는 표현이 모호해 후속 일정 관리가 어려울 수 있습니다.

6. 확인 필요 사항
- 리뉴얼의 구체적인 목적이 무엇인지 확인해야 합니다.
- 제품 페이지의 구성 범위와 우선순위를 확인해야 합니다.
- 클라이언트가 전달할 자료의 종류와 전달 기한을 확인해야 합니다.
- 문의 기능 포함 여부를 확정해야 합니다.
- 사이트맵 전달 가능 일정과 확정 여부를 확인해야 합니다.
- “다음 주쯤 정리”의 정확한 일정과 담당자를 확인해야 합니다.

7. 후속 일정
- 다음 주 중 후속 정리가 언급되었으나, 정확한 날짜와 담당자는 확인이 필요합니다.

추가 확인 질문
- 리뉴얼의 1순위 목적은 기업 신뢰도 개선, 제품 정보 전달, 문의 전환 중 무엇인가요?
- 제품 페이지에는 제품 목록, 상세 설명, 카탈로그 다운로드 중 어떤 요소가 필요한가요?
- 자료 전달 예정일은 언제인가요?
- 문의 기능은 필수 기능인가요, 검토 대상인가요?
- 사이트맵은 언제까지 전달 가능한가요?
- 다음 주 후속 정리는 내부 정리인지, 클라이언트 재회의인지 확인이 필요한가요?
- 참고할 레퍼런스 또는 벤치마킹 사이트가 있나요?
