# 프로젝트 1 - 자동화 도구 비교 구현 보고서

> Google Forms 기반 합격/불합격 자동 분류 시스템  
> **Make vs Zapier**

---

# 1. 프로젝트 개요

Google Forms 응답이 제출되면 점수에 따라 합격/불합격을 자동으로 판단하고, Google Sheets에 기록한 뒤 Gmail로 알림 메일을 발송하는 자동화 Workflow를 구현하였다.

동일한 Workflow를 **Make**와 **Zapier** 두 가지 도구로 각각 구현하고, 구성 방식과 특징을 비교 분석하였다.

## 공통 요구사항

|구분|구현 내용|
|---|---|
|Trigger|Google Forms 응답 제출|
|Action|Google Sheets 행 추가 + Gmail 메일 발송|
|조건 분기|점수 80점 기준 합격 / 불합격|
|실행 결과|합격·불합격 모두 정상 실행 확인|

---

# 2. 공통 Workflow 구조

```
Google Forms (응답 제출)
            │
            ▼
      조건 분기(Router / Paths)
      ┌─────────────────────┐
      │                     │
      ▼                     ▼
 합격(80점 이상)      불합격(80점 미만)
      │                     │
Google Sheets        Google Sheets
      │                     │
    Gmail               Gmail
```

---

# 3. Make 구현

## Workflow 구성

Google Forms(Watch Responses)를 Trigger로 사용하였다.

Router를 이용하여 점수를 기준으로 합격과 불합격 경로를 분기하였으며,

각 경로마다

- Google Sheets(Add a Row)
- Gmail(Send an email)

모듈을 연결하여 자동으로 데이터를 저장하고 메일을 발송하도록 구성하였다.

## Router 조건

### 합격 경로

```
평가점수 >= 80
```

### 불합격 경로

Fallback(대체 경로)

합격 조건을 만족하지 않는 모든 데이터가 자동으로 불합격 경로로 이동하도록 설정하였다.

---

# 4. Zapier 구현

## Workflow 구성

Google Forms(New Form Response)를 Trigger로 사용하였다.

Paths 기능을 이용하여

- Path A : 합격
- Path B : 불합격

으로 분기하였다.

각 Path마다

- Google Sheets(Create Spreadsheet Row)
- Gmail(Send Email)

Action을 연결하여 자동화를 구현하였다.

## Paths 조건

### Path A (합격)

```
평가점수 >= 80
```

### Path B (불합격)

```
평가점수 < 80
```

---

# 5. 실행 결과

## Make

- Google Forms 응답 정상 감지
- Router 조건 정상 분기
- Google Sheets 저장 완료
- Gmail 발송 완료

## Zapier

- Google Forms 응답 정상 감지
- Paths 조건 정상 분기
- Google Sheets(Create Spreadsheet Row) 실행
- Gmail(Send Email) 정상 발송

합격과 불합격 모두 정상적으로 실행되는 것을 확인하였다.

---

# 6. Make vs Zapier 비교

|비교 항목|Make|Zapier|
|---|---|---|
|UI|시각적 캔버스 방식|단계형(Setup → Configure → Test)|
|조건 분기|Router|Paths|
|학습 난이도|초기 적응 필요|초보자가 사용하기 쉬움|
|데이터 확인|모듈별 Output 확인|Data in / Data out 제공|
|무료 사용|멀티스텝 활용 가능|계정 상태에 따라 Paths 사용 가능|
|연동 앱|다양함|매우 다양함|

---

# 7. 장단점

## Make

### 장점

- 시각적으로 Workflow를 이해하기 쉽다.
- Router를 이용한 복잡한 분기에 강하다.
- 무료 플랜에서도 다양한 자동화가 가능하다.

### 단점

- 처음 사용하는 경우 캔버스 구조에 익숙해질 시간이 필요하다.

---

## Zapier

### 장점

- 단계별 설정 방식이라 직관적이다.
- Setup → Configure → Test 순서가 명확하다.
- 다양한 서비스와 연동 가능하다.

### 단점

- Task 기반 과금 정책으로 사용량이 많아지면 비용이 증가할 수 있다.
- 일부 기능은 계정 및 정책에 따라 제한될 수 있다.

---

# 8. 어떤 상황에 적합한가

### Make

- 복잡한 Workflow
- 다양한 조건 분기
- 시각적인 설계를 선호하는 경우

### Zapier

- 빠른 자동화 구축
- 비개발자 사용
- 다양한 SaaS 연동이 필요한 경우

---

# 9. 결론

동일한 자동화 Workflow를 Make와 Zapier 두 도구로 구현하면서 Trigger, Action, 조건 분기(Router / Paths)의 작동 원리를 직접 비교할 수 있었다.

Make는 시각적인 Workflow 구성과 Router 기반 분기에 강점을 보였으며,

Zapier는 단계별 마법사 형태의 설정과 직관적인 UI가 장점이었다.

두 도구 모두 Google Forms → 조건 분기 → Google Sheets → Gmail 자동화를 정상적으로 구현하였으며,

업무의 복잡도와 운영 환경에 따라 적합한 도구를 선택하는 것이 중요하다는 점을 확인하였다.
