# B2B 제조기업 홈페이지 리뉴얼 프로젝트 킥오프 회의록

### 1. 회의 목적
- 기존 홈페이지의 모바일 가독성 저하 및 제품 문의 전환율 낮음 문제 개선
- 1차 목표 수립: 회사 신뢰도 개선, 제품 정보 탐색 편의성 향상, 문의 접수 증가
- 주요 메뉴 구성 및 핵심 기능 범위 논의, 향후 일정 및 역할별 작업 조율

### 2. 결정사항
- **홈페이지 주요 메뉴 구성**: 회사소개, 제품소개, 기술자료, 납품사례, 고객문의
- **제품소개 페이지 구성**: 제품군별 목록, 상세페이지, 인증서 다운로드 영역 포함
- **고객문의 페이지 구성**: 이름, 연락처, 이메일, 문의 제품군, 문의 내용 입력 폼 구축
- **고객문의 연동 기능**: 문의 폼 접수 내역 이메일 발송 기능을 기본 범위에 포함
- **관리자 기능 1차 범위 조율**: 제품 정보 수정 기능만 우선 포함 (인증서 파일 관리 기능은 추가 견적 대상으로 분리)
- **퍼블리싱 작업 기준**: 반응형 기준으로 작업하되, 모바일 화면을 우선 고려

### 3. Action Items

| 담당 역할 | 작업 | 기한 |
| :--- | :--- | :--- |
| 클라이언트 담당자 | 인증서 관리 기능 포함 여부 내부 검토 후 회신 | 이번 주 금요일 |
| 클라이언트 담당자 | 회사 로고 원본 파일, 제품 카탈로그, 인증서 PDF, 납품사례 자료 전달 | 다음 주 월요일 |
| 기획 담당 | 사이트맵 및 주요 화면 구성안 작성 | 다음 주 화요일 |
| 프로젝트 매니저 | 기존 홈페이지 관리자 접속 정보 및 호스팅 정보 요청 | 확인 필요 |
| 디자인 담당 | 사이트맵 확정 후 메인 시안 1안 제작 | 확인 필요 |
| 퍼블리싱 담당 | 디자인 확정 후 반응형(모바일 우선) 퍼블리싱 작업 | 확인 필요 |

### 4. 확인된 사실
- 기존 홈페이지는 모바일 화면 가독성이 떨어지고 제품 문의 전환율이 낮은 상태임.
- 클라이언트는 관리자 페이지를 통해 제품 정보를 직접 수정하기를 원함.
- 개발 담당 의견에 따르면 제품 정보 관리 기능은 가능하나, 인증서 파일 업로드 관리까지 포함하면 개발 범위와 일정이 늘어날 수 있음.
- 기존 홈페이지의 게시판 데이터 양과 구조는 현재 확인되지 않아 데이터 이전 여부를 확정하지 못함.
- 다국어 페이지 필요성에 대한 의견은 나왔으나 구체적인 언어 범위와 번역 방식은 정해지지 않음.

### 5. 가능한 리스크
- 클라이언트의 자료(회사 로고, 카탈로그, 인증서, 납품사례 등) 전달이 지연되거나 인증서 관리 기능이 추가 범위로 확정될 경우, 최종 일정 및 개발 범위가 늘어날 수 있음.
- 기존 홈페이지 게시판 데이터의 양이 너무 많거나 구조가 복잡할 경우, 데이터 이전에 예기치 못한 리소스가 소요되거나 일정이 지연될 가능성이 있음.
- 다국어 페이지 도입 여부, 언어 범위, 번역 방식이 뒤늦게 확정될 경우 기획, 디자인, 개발 전 과정의 작업량이 증가하고 일정이 변동될 수 있음.

### 6. 확인 필요 사항
- **프로젝트 매니저의 기존 홈페이지 정보 요청 기한**: 관리자 접속 정보와 호스팅 정보를 클라이언트에게 언제까지 요청하고 언제까지 전달받을지 명확한 기한 확인 필요
- **디자인 및 퍼블리싱 세부 일정**: 사이트맵 확정 후 메인 시안 1안 제작 기한, 그리고 디자인 확정 후 퍼블리싱 완료 목표 기한 확인 필요
- **데이터 이전 여부**: 기존 홈페이지의 게시판 데이터 양과 구조 확인한 후 데이터 이전 진행 여부 및 구체적인 방식 확정 필요
- **다국어 페이지 범위**: 다국어 페이지 구축 여부, 필요한 언어의 범위 및 번역본 제공 방식 확인 필요
- **최종 오픈 일정**: 자료 전달 및 추가 기능 범위가 확정된 후, 다음 달 말 목표에 맞춘 최종 일정 재산정 필요

### 7. 후속 일정
- **이번 주 금요일**: 클라이언트의 인증서 관리 기능 포함 여부 회신
- **다음 주 월요일**: 클라이언트의 기본 자료(로고, 카탈로그, 인증서, 납품사례) 전달
- **다음 주 화요일**: 기획 담당의 사이트맵 및 주요 화면 구성안 작성 완료
- **이후 일정**: 사이트맵 초안이 나온 뒤 차기 회의 진행

6. **[최종 일정]** 자료 전달 및 추가 기능 범위 확정 후, 다음 달 말 오픈 목표를 바탕으로 한 최종 프로젝트 일정 재산정 필요
